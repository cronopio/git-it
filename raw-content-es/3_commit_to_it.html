El reto:

Crear un archivo en su nuevo repositorio, agregue algo al archivo y haga commit de los cambios a Git.

Paso: Cree un nuevo archivo

Ahora que el repositorio ha iniciado, agregue archivos a el.

Abra un editor de texto. Ahora escriba un par de lineas de texto, tal vez diga hola, guarde el archivo como readme.txt en el directorio 'hola-mundo' creado en el reto anterior.

Paso: Estado, Agregar y confirmar cambios

Comprobamos el estado de su repositorio para averiguar si hay cambios. Debajo en la terminal, usted debe estar dentro del directorio 'hola-mundo' que usted creo. Mira si hay cambios:

$ git status

Ahora agregue (add) el archivo que acaba de crear a los archivos que usted le gustaria confirmar (commit) (guardar) el cambio

$ git add readme.txt

Finalmente, confirme haciendo commit esos cambios al historial del repositorio con una descripcion corta sobre su actualizacion

$ git commit -m "<Su mensaje del commit>"

Paso: Haga m√°s cambios

Ahora agregue mas lineas al archivo readme.txt y guarde.

En la linea de comandos, usted puede ver la diferencia (diff) entre el archivo y como estaba en su commit anterior.

$ git diff

Ahora con lo aprendido arriba, haga un commit con estos ultimos cambios.

Verifique ejecutando

git-it verify

Pase al siguiente reto ejecutando

git-it

Ayuda!

  • Verifique el estado del repositorio
  • $ git status
  • Mire los cambios de un archivo
  • $ git diff
  • Agregar los cambios de un archivo para confirmar (commit)
  • $ git add <FILENAME>
  • Agregar todos los archivos modificados
  • $ git add .
  • Para hacer commit (confirmar) de los cambios agregados con un mensaje corto describiendo los cambios
  • $ git commit -m "<Su mensaje>"
